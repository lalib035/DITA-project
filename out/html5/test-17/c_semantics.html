Semantics in HTML

HTML has two types of elements: semantically meaningful elements and semantically meaningless elements. Proper use of these elements is essential to producing quality web content.

Semantic HTML refers to HTML elements that have meaning. This means that semantic elements don't just group content together—instead, semantic HTML elements also tell readers something about their contents. For example, the <nav> element tells users and their viewing devices that a particular group of links is the navigation bar. Semantically meaningless elements, like the generic containers <span> and <div>, are the rare HTML elements that don't automatically impart meaning on their contents. Generic containers simply group content together.

Before Cascading Style Sheets (CSS) existed, web writers had to use Hypertext Markup Language (HTML) elements to format documents. CSS today allows web writers to separate content (written in HTML) from presentation (written in CSS). Because many HTML elements are semantic (and thus describe their contents), writers who try to style text using HTML can accidentally change the meaning of the text at the same time.

An example of semantic element misuse is using the <em> element to italicize text in a reference section. <em> indicates that readers should emphasize a selection of text. Web browsers usually display the content of <em> elements as italicized text; however, there is a separate HTML element that should be used when web writers want to simply italicize, not emphasize, text: <i>. If a writer wanted a phrase to be italicized, and used the HTML element <em> to do so, he or she would also be telling readers to emphasize the text selection as they read. Instead of just formatting a references section to follow APA style by italicizing book titles, authors would be telling readers to read book titles differently. This misuse of elements becomes especially important when browsers render elements differently and users surf the web using screen readers.

Generic containers are one way authors can avoid misusing semantic elements. If there isn't an element that fits the author's need, like an element that adds extra formatting to the first two paragraphs of text, authors can use generic containers—in this case, <div>—to group the desired content together so CSS presentation rules can be applied.
